# Arduino Sensors and Modules Categorized Cheatsheet

This cheatsheet organizes sensors and modules for Arduino by category (e.g., Environmental Sensing, Motion Sensing, Communication, Display, Actuators, Storage) for quick reference. It details their purpose, use cases, specifications, libraries, and example code snippets. Designed for engineering students working on embedded systems.

## Selection Guide
- **Environmental Sensing**: DHT11 (budget), BME280 (precision), MQ-2 (gas), LDR (light).
- **Motion/Proximity**: HC-SR04 (distance), PIR (motion), MPU6050 (orientation), IR (line/proximity).
- **Communication**: HC-05 (short-range Bluetooth), ESP8266 (WiFi/IoT).
- **Display**: LCD (text), OLED (graphics).
- **Actuators**: Servo (angular), Stepper (steps), Relay (power switching).
- **Storage**: SD Card (large data), EEPROM (small persistent data).

## Quick Tips
- **Power**: Match 5V/3.3V requirements. Use external power for actuators (e.g., motors).
- **Pin Conflicts**: Avoid Serial pins (0, 1). Use PWM pins (3, 5, 6, 9, 10, 11 on Uno) for `analogWrite`.
- **Libraries**: Install via Arduino IDE Library Manager. Include `#include <Library.h>`.
- **Debugging**: Monitor sensor values with `Serial.println()`.
- **Interfacing**: Use I2C/SPI for complex modules, analog/digital for simple sensors.

## Example: Motion-Triggered Relay
```c
#define pirPin 2
#define relayPin 7
void setup() {
    pinMode(pirPin, INPUT);
    pinMode(relayPin, OUTPUT);
}
void loop() {
    if (digitalRead(pirPin) == HIGH) {
        digitalWrite(relayPin, HIGH); // Activate relay
    } else {
        digitalWrite(relayPin, LOW);
    }
}
```
